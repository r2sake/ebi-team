# 🦐 ebi-team をスマホから使う（セットアップ手順）

外出先（自宅の Wi-Fi の外でも）スマホから ebi-team の Web UI を開いて、エビの様子を見たり操作したりできるようにする手順です。**インフラの知識は不要**。指示どおり順にやれば動きます。

## この方式の考え方（安心のために一言だけ）

ebi-team の画面は `claude` を丸ごと操作できる強力な画面です。だから「インターネットに直接は絶対に公開しない」のが鉄則です。この手順では **2 重の鍵**で守ります。

1. **Tailscale**（テールスケール）＝「自分のデバイスだけが入れる専用の細い道」を作るアプリ。Mac とスマホを同じ“身内ネットワーク”に入れます。他人は物理的に到達できません。
2. **アクセストークン**＝ ebi-team 自身のパスワード。上の道に入れた身内でも、トークンを知らなければ画面は開けません。

この 2 つが揃うので安全です。以下、① Mac 側 → ② スマホ側 の順に進めます。

---

## ① Mac 側の準備

### 1-1. Tailscale を入れる

1. https://tailscale.com/download/mac を開き、Tailscale アプリをダウンロードして入れる（App Store 版でも可）。
2. 起動して **Log in**。Google などのアカウントでログインします（**スマホでも同じアカウントを使う**のがポイント。これで同じ身内ネットワークになります）。
3. メニューバーに Tailscale アイコンが出れば OK。アイコンをクリックすると、この Mac の名前（例: `mac-mini`）と IP（`100.x.y.z`）が確認できます。**この Mac の名前をメモ**しておいてください（あとでスマホから使います）。

### 1-2. アクセストークンを決める

ターミナルで次を実行すると、ランダムで十分に強いトークンが 1 個出ます。

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

出てきた 64 文字の文字列をコピーしてメモしておきます（例: `3f9a...c1`）。**これがスマホで入力するパスワードになります。**

### 1-3. ebi-team を「トークンあり・Tailscale 経由で待受」で起動する

まずビルド（初回や更新後のみ）:

```bash
npm run build
```

設定は **リポジトリルートの `.env` ファイル**に書きます。まず雛形をコピー:

```bash
cp .env.sample .env
```

`.env` をエディタで開き、次の 2 つの行のコメント（先頭の `#`）を外して値を入れます。

- `EBI_AUTH_TOKEN` … さっき決めたトークン
- `EBI_HOST` … この Mac の **Tailscale の IP**（`100.x.y.z`）。Tailscale アイコンから確認できます。

```dotenv
EBI_AUTH_TOKEN=ここにさっきのトークン
EBI_HOST=100.x.y.z
```

あとは起動するだけ:

```bash
npm start
```

> **環境変数を直接付けても動きます。** `.env` を使わず
> `EBI_AUTH_TOKEN=... EBI_HOST=100.x.y.z npm start` のように一行で渡してもOKです
> （その場合、直接指定した値が `.env` より優先されます）。

> **なぜ `EBI_HOST` に Tailscale の IP を入れるの？**
> こうすると「Tailscale の道からしか受け付けない」状態になり、家の Wi-Fi の他の機器からも直接は見えなくなります（いちばん安全）。とりあえず動かしたいだけなら `EBI_HOST=0.0.0.0` でも可ですが、その場合でもトークンで守られます。

起動ログに次の 1 行が出ていれば、トークン認証が効いています:

```
[ebi-team] 認証: EBI_AUTH_TOKEN 設定あり（非 loopback はトークン必須・/login で入力）
```

もし `EBI_AUTH_TOKEN 未設定のまま非 loopback に bind` という**警告**が出ていたら、トークンを付け忘れています。外部からのアクセスは全部はじかれる（安全側）ので害はありませんが、スマホから入れません。付け直してください。

> Mac 上のブラウザからは今までどおり `http://localhost:8787`（トークン不要）で開けます。ローカルの使い勝手は変わりません。

## ② スマホ側の準備（iPhone / Android 共通）

### 2-1. Tailscale アプリを入れる

1. App Store / Google Play で **Tailscale** を入れる。
2. 起動して **Mac と同じアカウント**でログイン。
3. 接続を ON にする（トグル / VPN の許可を求められたら許可）。これで Mac と同じ身内ネットワークに入りました。

### 2-2. ブラウザで開く

Safari / Chrome で次のアドレスを開きます（`<Mac名>` は 1-1 でメモした Mac の名前）:

```
http://<Mac名>:8787
```

例: `http://mac-mini:8787`

> 名前でうまく開けないときは、Tailscale の IP でも開けます: `http://100.x.y.z:8787`

### 2-3. トークンを入力

ログイン画面が出るので、**1-2 で決めたトークン**を貼り付けて「ログイン」。
一度入れれば端末に記憶される（Cookie）ので、次回からは入力不要でそのまま画面が出ます。

これで完了です。エビ一覧が見え、ペインをタップすれば入力もできます。

### スマホでの操作のコツ

- 右上の **☰** ボタンで registry（エビ一覧）の開閉。エビを選ぶと自動で閉じてメイン画面に戻ります。
- 画面下の **入力補助バー**（Esc / Tab / ↑↓←→ / Ctrl / ^C）で、スマホのキーボードに無いキーを送れます。
  - **Ctrl** を押すと光り、その直後に押した 1 文字が Ctrl 付きになります（例: Ctrl → `c` で Ctrl+C）。^C ボタンは Ctrl+C の直接送信です。
- ペインをタップするとソフトキーボードが出ます。

## トラブルシュート（つながらない時の確認）

順番にチェックしてください。

1. **スマホの Tailscale が ON になっているか。** 一番よくある原因です。アプリを開いて接続中か確認。
2. **Mac と同じアカウントで Tailscale にログインしているか。** 別アカウントだと同じ身内ネットワークに入れません。
3. **Mac 側で `npm start` が動いているか。** ターミナルを閉じると止まります。起動ログが出たままか確認。
4. **アドレスは合っているか。** `http://<Mac名>:8787`。`https` ではなく `http`、ポート `8787`。名前でダメなら `http://100.x.y.z:8787`（Tailscale の IP）を試す。
5. **ログイン画面は出るがトークンで弾かれる。** 1-2 でコピーしたトークンと、起動時に `EBI_AUTH_TOKEN=` に渡した値が一致しているか確認（前後の空白やコピー漏れに注意）。
6. **「試行回数が多すぎます」と出た。** 短時間に何度も失敗すると一時的にブロックされます。1 分ほど待って再試行してください。
7. **画面は開くが真っ黒／エビが出ない。** WebSocket 接続の問題の可能性。ページを再読み込み。それでもダメなら Mac 側のターミナルのログにエラーが出ていないか確認。
8. **Mac を再起動した後。** Tailscale と `npm start` を起動し直す必要があります（自動では立ち上がりません）。

## 補足（技術メモ）

- 通信は Tailscale（WireGuard）で暗号化されるため、平文 `http` でも経路上は保護されます。ブラウザの「保護されていない通信」警告が気になる場合は、将来 `tailscale cert` / `tailscale serve` で TLS を付けられます（任意・Phase2）。
- トークンは母艦ローカル（`localhost` / `127.0.0.1`）からのアクセスには要求されません。エビ同士の内部連携（MCP 制御）や Mac 上のローカル利用はこれまでどおり無認証で動きます。
- `EBI_AUTH_TOKEN` を設定しないまま外向き（非 loopback）に bind した場合、外部アクセスは**すべて拒否**されます（事故防止の安全側デフォルト）。
- 使うポートを変えたいときは `EBI_PORT=別番号` を付けて起動し、スマホのアドレスもそのポートに合わせます。
